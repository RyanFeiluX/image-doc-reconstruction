## 功能说明
本技能采用两遍遍历机制，智能探测并过滤文档中的页眉、页脚、分隔线、Logo等辅助元素，确保输出文档的纯净度和结构完整性。

## 工作原理

### 第一遍：模式探测
```
遍历所有页面
├─ 使用普通模式OCR（保持文档结构）
├─ 统计每个位置（center_x, center_y）出现的内容
├─ 识别重复出现的内容（≥3次）
├─ 分析位置分布：
│  ├─ 页眉区域（顶部12%）
│  └─ 页脚区域（底部12%）
└─ 生成页眉页脚模式列表（位置指纹）
```

### 第二遍：实际处理
```
遍历所有页面
├─ 使用普通模式OCR（保持文档结构）
├─ 应用页眉页脚过滤：
│  ├─ 检查block是否匹配页眉页脚模式
│  ├─ 匹配则过滤，不匹配则保留
│  └─ 支持位置指纹模糊匹配（容差5像素）
├─ 应用辅助元素过滤（分隔线、Logo、关键词）
├─ 基于过滤后的blocks进行布局分析
├─ 字体统计（不受页眉页脚影响）
└─ 生成Markdown
```

## 探测算法

### 位置指纹
```python
position_key = f"{center_x:.0f}_{center_y:.0f}"
```
- 将文本块的中心点坐标作为位置指纹
- 精度：像素级（取整）
- 支持模糊匹配：容差5像素

### 重复检测
- 统计每个位置指纹出现的次数
- 出现次数 ≥ 3次 → 标记为页眉页脚模式
- 适应不同长度的文档

### 区域判断
```python
if center_y < page_height * 0.12:
    region = 'header'  # 顶部12%
elif center_y > page_height * 0.88:
    region = 'footer'  # 底部12%
else:
    region = 'body'
```

## 可识别的元素类型

| 类型 | 检测策略 | 示例 |
|------|---------|------|
| **页眉** | 位置指纹 + 重复检测 | 公司名称、文档标题、日期 |
| **页脚** | 位置指纹 + 重复检测 | 页码、版权信息、网址 |
| **分隔线** | 全局，单字符重复 | `=====`, `-----`, `_____` |
| **页码** | 页脚区域 + 页码模式 | `1`, `Page 1`, `- 1 -` |
| **Logo** | 角落区域 + 短文本 | 页面左上/右上角的Logo区域 |
| **页眉关键词** | 页眉区域 + 特定关键词 | `Confidential`, `Draft`, `保密` |
| **页脚关键词** | 页脚区域 + 特定关键词 | `©`, `All rights reserved`, `www.` |

## OCR模式自动切换

| 场景 | OCR模式 | 理由 |
|------|---------|------|
| **启用辅助元素过滤（默认）** | 普通模式 | 保持文档结构，页眉页脚位置清晰 |
| **禁用辅助元素过滤** | 高精度模式 | 追求最高识别精度 |
| **强制使用高精度** | 高精度模式 | 用户显式指定 |

**普通模式 vs 高精度模式对比**：
- **普通模式**：速度快，结构保持好，文本块数量适中
- **高精度模式**：识别精度高，但可能过度分割，影响结构判断

## 配置说明（推荐默认值）

| 配置项 | 默认值 | 说明 |
|------|--------|------|
| `filter_auxiliary` | `True` | 启用页眉页脚探测与过滤（推荐） |
| `accurate_ocr` | `None` | 自动选择OCR模式（推荐） |
| `include_figures` | `True` | 包含插图（推荐） |
| `embed_figures` | `True` | 嵌入图片到Markdown（默认，便于分享） |
